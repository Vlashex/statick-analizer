# Статический анализатор сложности C++

Этот каталог фиксирует документацию MVP проекта. Документы ниже задают минимальный рабочий каркас для реализации учебного анализатора.

## Состав документации

- [Спецификация](./spec.md) — исходные функциональные требования и ограничения.
- [Архитектура](./architecture.md) — минимальная архитектура MVP, pipeline и доменная модель.

## Назначение системы

Система предназначена для статического анализа временной сложности функций на C++ без выполнения исходного кода. MVP сфокусирован на базовых конструкциях:

- циклы `for`, `while`, `do-while`
- ветвления `if`, `else`, `switch`
- базовая блочная структура и простые операторы

## Границы первой версии

- На вход подается путь к одному `.cpp` файлу.
- На выходе печатается отчёт по функциям в консоль: имя и оценка сложности.
- Анализатор работает только с поддерживаемым подмножеством C++ и не моделирует весь язык.
- Визуализация, API, БД и внешние сервисы в MVP не входят.

## Принципы реализации

- Архитектура минимальная: `CLI -> lexer -> parser -> analyzer -> console`.
- Используются только простые модули, необходимые для текущего CLI MVP.
- Контракты модулей строгие: фиксированные вход/выход и отсутствие прямых зависимостей между внутренними частями модулей.
