# Функциональные требования для статического анализатора сложности кода на C++

## Введение

Статический анализатор сложности кода на C++ — это инструмент, предназначенный для вычисления временной сложности функций в исходных кодах на языке C++ без их выполнения. Анализатор работает с текстом исходного кода и оценивает его сложность на основе структуры и логики кода, используя типовые модели сложности для базовых конструкций языка.

## 1. Входные данные

1.1. Программа получает на вход текст исходного кода на языке C++ в виде строкового ввода или через чтение из файла (формат: .cpp).

1.2. Код должен содержать следующие элементы:

- Функции и методы
- Условия (`if`, `else`, `switch`)
- Циклы (`for`, `while`, `do-while`)
- Вызовы функций
- Логические и арифметические операторы

## 2. Выходные данные

2.1. Программа выводит для каждой функции:

- Имя функции
- Оценка временной сложности функции (например, O(n), O(n^2), O(log n) и т. п.)
- Причины оценки сложности для каждой конструкции внутри функции (например, для циклов, рекурсии, ветвлений).

## 3. Основные функции

3.1. **Анализ циклов**  
Для каждой конструкции цикла программа должна определять его вклад в общую сложность:

- Цикл `for`, `while`, `do-while`: анализ количества итераций и операций внутри цикла.
- Вложенные циклы должны учитывать степень вложенности и множить их сложности.

  3.2. **Анализ условий**  
  Программа должна оценивать влияние условий на сложность:

- Оценка влияния ветвлений на количество выполняемых операций в зависимости от входных данных.

  3.3. **Анализ рекурсии**  
  Для рекурсивных функций программа должна использовать стандартные методы для анализа рекурсивных вызовов (например, метод подстановки для вычисления сложности).

  3.4. **Анализ вызовов функций**  
  Программа должна учитывать вызовы других функций и их сложности, если они известны. Если функция вызвана несколько раз, это должно учитываться в общей сложности.

## 4. Ограничения

4.1. Программа не должна анализировать весь C++, а только базовые конструкции:

- Операторы ветвления (условия, переключения)
- Операторы циклов (for, while)
- Вызовы функций (плоские или рекурсивные)

  4.2. Программа не должна реализовывать визуализацию или вывод графиков.

  4.3. Программа не должна анализировать сторонние библиотеки или системные функции, если их сложность не указана в коде.

## 6. Заключение

Статический анализатор сложности кода на C++ должен быть простым в реализации и выполнять базовый анализ временной сложности на основе ключевых конструкций языка. Это позволит пользователю получить оценку производительности для функций, что может помочь в оптимизации и улучшении кода.
